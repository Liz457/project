Git

Git is one of the most popular version control systems. version control keeps track of every change in your code, and it allows you to go back in time when something goes wrong. Also, it is really helpful to prevent concurrent work from conflicting when multiple people are working in the same project. An individual may be working on the sidebar navigation while another one is simultaneously updating the header.

Branches:

A branch is a new/separate version of the main repository. Branches allow you to work on different parts of a project without impacting the main branch. When the work is complete, a branch can be merged with the main project.

You can even switch between branches and work on different projects without them interfering with each other. Branching in Git is very lightweight and fast. Checkout is the command used to check out a branch. Moving us from the current branch, to the one specified at the end of the command.

Merge:

The git merge command lets you take the independent lines of development created by git branch and integrate them into a single branch. Git merge will combine multiple sequences of commits into one unified history. In the most frequent use cases, git merge is used to combine two branches. Git can automatically merge commits unless there are changes that conflict in both commit sequences.

Commit:

Saves the group of changes you staged to your local repository. Add a comment with each commit that summarizes the change. Git commit command is the main function for saving changes to local repository safely. Commit takes a snapshot of the project and treats it as a version. Git does not add changes to a commit automatically. You need to indicate which file and changes need to be saved before running the Git commit command. The commit command does not save changes in remote servers, only in the local repository of Git. After running the Git commit command, you need to type in the description of the commit in the text editor. Once you move the changes to the local repository by using Git commit command, you can use Git push to transfer them to a remote repository.

Tags:

Tags are ref's that point to specific points in Git history. Tagging is generally used to capture a point in history that is used for a marked version release. A tag is like a branch that doesn’t change. The git tag command is the primary driver of tag: creation, modification and deletion.

Repositories:

Git repository serves as a storage for all the changes and files that relate to a certain project. Initializing Git repository is quite simple. Some Git repositories can be local, placed directly at your local computer. You can use an already existing directory as a repository for your Git files or create a brand new one. If you work in a team or you are invited to make changes to a particular code, chances are you will need to access a remote Git repository. Git clone command or Git clone repository are the names for a command that create a local version for you of that remote Git repository so that you could create your own changes without any damage to the remote version.

Git Commands:

  1. Create a new branch: git branch branch_name
  2. Checkout a branch: git checkout branch_name
  3. Check the status of the current branch: git status
  4. Commit changes to the branch: git commit -m "new changes"
  5. Initialize an existing directory as a Git repository:git init
  6. Retrieve an entire repository from a hosted location via URL: git clone [url]
  7. Merge the specified branch’s history into the current one: git merge [branch]
  8. Show all commits in the current branch’s history: git log
  9. Transmit local branch commits to the remote repository branch: git push [alias] [branch]
  10. Apply any commits of current branch ahead of specified one: git rebase [branch]
  11. Fetch and merge any commits from the tracking remote branch: git pull
