S.O.L.I.D.

  1. S: Single-Responsibility Principle
    In the Calculator class, all the methods are following the SOLID principle. For Example, The add method adds 2 different values and gives the result. The subtract methods subtracts the one value from another. So, the single responsibility of the division method is to divide two numbers, store the result of the division calculation in the results and return it.
  2. O: Open-Closed Principle
    This principle explains that the method should do one thing, but it should also be complete whatever problems it might get. For example have error handlers. That way the function will finish the job whatever it is designed to do. The Calculator satisfies the SOLID principle in this method as well.
  3. L: Liskov Substitution Principle
    According to this principle, any subtype class should be 100% compatible with its base class. The Calculator satisfies this principle. The Calculator class is 100% compatible for it's base classes.
  4. I: Interface Segregation Principle
    This principle facilitates inheritance. Class should not be forced to depend on those of its methods that they donâ€™t use. Class should only see the methods they will use, and subclasses should inherit only the methods they need. Otherwise, structures must be dismantled, avoid fat classes.In this Calculator program, the classes are only using what they need.
  5. D: Dependency inversion Principle
    This principle explains that High level modules should not depend upon low level modules. Both should depend upon abstractions. Abstractions should not depend upon details, details should depend upon abstractions.In the Calculator program, there is no dependency between instance and its method.
